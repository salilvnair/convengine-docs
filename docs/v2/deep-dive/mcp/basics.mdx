# Agent Tool Basics

This page gives the simple mental model for Agent Tools and MCP in ConvEngine.

<Highlight type="info" title="Naming clarification">
**Agent Tools** = ConvEngine's own DB-driven tool capabilities (ce_agent_tool table).  
**MCP Servers** = external servers connected via the real MCP protocol (JSON-RPC 2.0, discovered at runtime).  
Both are surfaced to `AgentPlanner` — `CALL_TOOL` works for either without any difference in planner behaviour.
</Highlight>

## What the agent execution loop does

1. User sends a message
2. `AgentPlanner` receives the user input, available Agent Tools, runtime-discovered MCP tools, and prior observations
3. Planner chooses `CALL_TOOL` or `ANSWER`
4. Tool output is stored in `context.agent.observations`
5. Planner loops until it returns `ANSWER`
6. `context.agent.finalAnswer` is written and response is resolved

## Real MCP Server tools

If any external MCP servers are registered via `McpRegistry`, their tools are discovered automatically and merged into the planner's tool list at runtime. No `ce_agent_tool` entry is needed — the planner sees them alongside DB-registered Agent Tools.

Tool code format: `mcp.{serverId}.{toolName}` (e.g. `mcp.ck8t-server.generate_component`)

Routing: `McpServerToolExecutor` handles tool group `MCP_SERVER`, parses the tool code, and calls `McpRegistry.callTool()`.
