# Agent Tools & MCP

ConvEngine has two distinct but complementary concepts that are often discussed together:

<Highlight type="info" title="Two separate things — same pipeline">
**Agent Tools** are ConvEngine's own DB-registered capabilities (DB queries, HTTP APIs, workflows, etc.).  
**MCP Servers** are external servers connected via the real MCP protocol (JSON-RPC 2.0).  
The `AgentPlanner` treats both transparently — `CALL_TOOL` works for either.
</Highlight>

## Agent Tools

Agent Tools are ConvEngine's own capability registry. Each tool is defined in `ce_agent_tool` — scoped to an intent and state — and executed by a typed executor inside the framework. The `AgentPlanner` selects which tool to call based on the user query and prior observations.

Tool groups supported out of the box:

| Group | Executor | Purpose |
|---|---|---|
| `DB` | `DbToolExecutor` | SQL queries with guardrails and preflight |
| `HTTP_API` | `HttpApiToolExecutor` | REST / SOAP calls with retry and auth |
| `WORKFLOW_ACTION` | `WorkflowActionToolExecutor` | Trigger workflow actions |
| `DOCUMENT_RETRIEVAL` | `DocumentRetrievalToolExecutor` | RAG / document search |
| `CALCULATOR_TRANSFORM` | `CalculatorTransformToolExecutor` | Math and data transforms |
| `NOTIFICATION` | `NotificationToolExecutor` | Email / SMS / Slack |
| `FILES` | `FileToolExecutor` | File I/O |
| `MCP_SERVER` | `McpServerToolExecutor` | External MCP server tools (see below) |

## Real MCP Servers

ConvEngine connects to any external MCP server (STDIO, HTTP, SSE transport) via `McpRegistry`. Servers are registered via the REST API — tools are discovered at runtime via `discoveredTools()` and surfaced to `AgentPlanner` alongside DB-registered Agent Tools. No DB entry needed for external MCP tools.

Tool code format for discovered tools: `mcp.{serverId}.{toolName}`
